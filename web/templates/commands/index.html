How Commands Are Sent via LoRa

What is this interface? Dispatch pre-compiled task scripts to remote buoy nodes (fish detection, battery reports, sonar profiles). For raw shell access use the Linux Console from the sidebar.

Why is there a delay? LoRa duty-cycle limits (≈1%) mean packets are queued and sent during the node's listening window. An ACK confirms receipt; CMD_STATUS follows on task completion.

Command Docs

Preset Scripts: Pre-installed on each node. They execute quickly with minimal payload sizes, reducing LoRa collision probability compared to raw Linux commands.

Troubleshooting

Queued Status: If a command stays "QUEUED" or "PENDING" for more than 2 minutes, check the RSSI strength of the node — it may have missed its wake window or the radio link is degraded.

Glossary

ACK (Acknowledgement)
A small packet returned by the node confirming it successfully demodulated the command packet.
Exit Code
A status code from a process. 0 = normal completion; non-zero (1, 127…) = execution error.
