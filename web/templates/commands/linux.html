System Architecture

How Shell Access Works Without Internet

What is this console? This page emulates an SSH connection to run command line strings on the remote buoy node's operating system (typically running Debian/Ubuntu Linux on a Raspberry Pi or BeagleBone).

How is it bridged? Standard SSH requires a continuous TCP socket, which is impossible over LoRa. Instead, we package the shell command into a single LoRa frame. The node's background daemon listens for the command, executes it using Python's subprocess module, wraps stdout/stderr output, chunks the results, and sends them back in packets. This provides offline administration at the expense of terminal interactivity.

Console Docs

    
Subprocess execution: Commands are executed inside a subshell with a 15-second timeout constraint to prevent scripts from hanging and exhausting node battery capacity.





    
Security & Safety

    
Command Filtering: Destructive commands like rm -rf /, reboot loop scripts, or persistent packet sniffers are automatically blocked by the node's payload security filter.





    
Glossary

    

        
Exit Code 0

        
Indicates success. The subprocess executed with no errors.

        
        
Exit Code 127

        
Indicates command not found. You typed a utility name that is not installed on the node's shell search path.
